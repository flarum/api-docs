protected HtmlResponse authenticate(array $identification, array $suggestions = [])

Respond with JavaScript to inform the Flarum app about the user's authentication status.

An array of identification attributes must be passed as the first argument. These are checked against existing user accounts; if a match is found, then the user is authenticated and logged into that account via cookie. The Flarum app will then simply refresh the page.</p> <p>If no matching account is found, then an AuthToken will be generated to store the identification attributes. This token, along with an optional array of suggestions, will be passed into the Flarum app's sign up modal. This results in the user not having to choose a password. When they complete their registration, the identification attributes will be set on their new user account.
